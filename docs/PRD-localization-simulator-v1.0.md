# PRD: Localization Simulator
**状态**: 已批准  **作者**: Andy (AI PM)  **最后更新**: 2026-04-09  **版本**: 1.0
**相关方**: Lisa (UX), Max (Dev), Tom (QA), Ray (DevSecOps)

---

## 1. 问题陈述
医疗器械企业在进入新市场时，需要经历复杂的本地化注册流程（设计转移 → 型号测试 → 注册申报审批 → 标签放行发货）。每个阶段包含多个 Scenario，各有不同周期。

**核心痛点**：RA（Regulatory Affairs）团队的注册规划数据通常以 Excel 表格形式存在，非 RA 背景的高管团队难以快速理解整体时间线、关键路径和资源投入，导致投资决策延迟或基于不完整信息做出。

**目标用户**：
- **主要用户**：RA Manager / Specialist — 录入和维护注册规划数据
- **核心受众**：VP / SVP / C-Suite — 查看可视化，做出 Go/No-Go 和优先级决策

---

## 2. 目标与成功指标
| 目标 | 指标 | 目标值 |
|------|------|--------|
| 高管理解效率 | 首次查看可视化后能说出关键路径 | < 2 分钟 |
| 数据录入效率 | 完成一个完整项目录入 | < 10 分钟 |
| 决策支持 | 可视化包含做 Go/No-Go 决策所需全部信息 | 100% |

---

## 3. 非目标（Non-Goals）
- 不做多用户协作和权限管理（v1 为单机 Web 应用）
- 不做与外部系统（SAP, Oracle）的数据集成
- 不做审批工作流或状态跟踪
- 不做成本/费用计算（v2 考虑）

---

## 4. 设计决策（Andy 对澄清问题的决策）

| 问题 | 决策 | 理由 |
|------|------|------|
| 投资决策类型 | Go/No-Go + 多市场时间线对比 | 最常见的高管决策场景 |
| Scenario 关系 | 同阶段内并行，阶段间串行 | 符合注册流程实际：测试可并行，但阶段有先后依赖 |
| 技术形态 | 单页 Web 应用（SPA），可部署为静态站点 | 零后端依赖，易分享，可离线使用 |
| 可视化形式 | 甘特图/时间线 + 摘要仪表盘 | 甘特图展示流程，仪表盘展示总时长和关键数字 |
| 额外数据维度 | MVP 只含 Description + Turnover Time | 保持简洁，v2 加费用和风险 |
| 现有工具 | 假设替代 Excel 工作流 | 最常见的现状 |

## 5. 用户故事

**US-1**: 作为 RA Manager，我希望在 Intake Sheet 中按阶段录入 Scenario 信息，以便系统自动生成时间线。
**验收标准**:
- [ ] 4 个阶段各自可添加/删除 Scenario
- [ ] 每个 Scenario 包含 Description（文本）和 Turnover Time（天数）
- [ ] 输入即时保存到浏览器本地存储

**US-2**: 作为高管，我希望看到一个清晰的 Gantt 时间线，展示从设计转移到标签放行的全流程。
**验收标准**:
- [ ] 阶段间串行显示，同阶段内 Scenario 并行显示
- [ ] 每个阶段的总时长 = 该阶段内最长 Scenario 的 Turnover Time
- [ ] 总项目时长 = 各阶段总时长之和
- [ ] 颜色编码区分不同阶段

**US-3**: 作为高管，我希望看到摘要仪表盘，一眼了解关键数字。
**验收标准**:
- [ ] 显示总项目周期（天/周/月）
- [ ] 显示各阶段周期及占比
- [ ] 显示 Scenario 总数
- [ ] 高亮关键路径（最长的 Scenario）

**US-4**: 作为用户，我希望可以导出可视化结果用于会议演示。
**验收标准**:
- [ ] 支持导出为 PNG 图片

## 6. 技术方案概述
- **框架**: React 18 + TypeScript + Vite
- **可视化**: Recharts（甘特图/条形图）
- **样式**: CSS Variables（Lisa 的 Design Token 系统）
- **存储**: localStorage（无后端）
- **导出**: html2canvas 生成 PNG
- **部署**: 静态站点（GitHub Pages / Vercel）
